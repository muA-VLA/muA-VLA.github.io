Abstract

From brittle chained execution to recoverable long-horizon behavior

Recent advances in vision-language-action (VLA) models have substantially improved robotic manipulation, yet they remain brittle in long-horizon, multi-step tasks where errors accumulate through chained execution, often leading to deadlocks and premature termination.

To address this limitation, we propose µA-VLA, a multi-agent online self-healing framework for long-horizon robotic manipulation. µA-VLA organizes three collaborative agents into a reasoning-coding-intervention pipeline. The Vision Failure Reasoning Agent analyzes failure visual observations and records the key causes in Failure Analysis Records. The Code Synthesis Agent combines these records with the Failure-Type Solution Library and the Task Success Rates Records to dynamically synthesize runnable code. This code instantiates a downstream Task-Specific Intervention Agent, which produces intervention instructions to guide the underlying VLA policy and reliably advance task progress.

Extensive evaluations demonstrate that µA-VLA, which leverages π0.5 as the execution backbone, achieves new state-of-the-art performance on the CALVIN ABC→D benchmark, reaching an average sequence length of 4.76 with a 90.1% success rate, alongside robust real-world physical deployments.

Framework

A reasoning-coding-intervention loop

The framework starts from the task instruction, previous subtask context, and failure visual observation. It then links failure diagnosis, code synthesis, and intervention execution into a closed loop.

Reason

Vision Failure Reasoning Agent

Infers root causes from visual observations, task context, and previous subtask state.

Code

Code Synthesis Agent

Combines failure records with reusable solution knowledge to synthesize executable intervention code.

Intervene

Task-Specific Intervention Agent

Issues corrective instructions to the underlying VLA controller and writes execution outcomes back to memory.

Real-world Deployment

Recovering from physical bottlenecks during execution

On a 6-DOF Synria Alicia-D robotic arm with RGB and wrist-camera observations, µA-VLA diagnoses long-horizon failures and inserts corrective actions before continuing the original task sequence.

Where recovery matters most

In difficult sequences, the baseline can become stuck because a key precondition is missing, such as attempting to place an object into a drawer that is not open. µA-VLA reasons about the missing condition online and calls the underlying VLA to perform the corrective action first.

In well-conditioned orders where preconditions are already satisfied, the backbone may complete the chain without intervention. This helps clarify that µA-VLA is most useful when long-horizon execution creates hidden bottlenecks.
